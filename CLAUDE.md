# CLAUDE.md — session conventions for `wildfire-exposure-eo`

> Read this file before doing anything. Every Claude Code session in this repository operates under the constraints below. Violations should be flagged in the session, not silently worked around.

## Mission, in one sentence

Score OSM-derived critical infrastructure for wildfire exposure on a Portuguese pilot AOI, using public STAC-native EO data, with full per-asset provenance, validated against ICNF burned-area history.

If a proposed action does not advance this mission, say so before doing it.

## Non-negotiables

These are conditions every artifact must satisfy. Violating any of them is grounds for rejecting the change without further review.

1. **No invented identifiers.** Never fabricate STAC item IDs, OSM IDs, model names, AOI bboxes, dataset DOIs, or paper citations. If a real value isn't on hand, query for it; if querying isn't possible right now, leave a `# TODO(provenance):` marker and surface it in the session summary.
2. **CRS is explicit, always.** Every raster and vector loaded, written, or compared must have a documented CRS. Implicit reprojection is forbidden. Use `rio.set_crs` and `gdf.set_crs` explicitly. Tests assert CRS on every fixture.
3. **Provenance is mandatory.** Every scored asset row must carry the full provenance dict defined in `README.md`. A row missing `model_version`, `run_id`, `code_commit_sha`, or any source STAC ID is a bug, not a feature.
4. **Determinism by default.** Default random seed is `42`. Any function that uses RNG must accept a `seed` argument and pass it to all sub-calls. Set `PYTHONHASHSEED=0` in `Makefile` and CI.
5. **Outputs are STAC-native or GeoParquet.** No Shapefile, no GeoPackage, no pickled dataframes, no custom binary formats as final outputs. COG for rasters, GeoParquet for vectors, JSON for STAC.
6. **No probability claims.** The exposure score is a relative rank, not a probability. README, code comments, docstrings, and validation report must all use *exposure*, *rank*, *relative*, *normalized* language. Never *risk probability*, *chance of fire*, or anything implying calibrated forecast.
7. **No live PII or operator-private data.** OSM is fine. ICNF public is fine. Do not pull e-Redes, REN, or DGEG private datasets even if a path appears available.
8. **No new dependencies without justification.** Every addition to `pyproject.toml` requires (a) a one-line justification in the PR description and (b) a pinned version. Prefer the existing stack.
9. **No production claims.** This is a public-data demonstrator. Words like *production-ready*, *utility-grade*, *operationally validated* are forbidden in README, comments, and commits.
10. **AOI is frozen.** Read from `data/aoi/pilot.geojson` everywhere. Never hardcode coordinates in source code.

## Required commands before declaring a task complete

In order. All four must pass.

```bash
uv run ruff check .
uv run ruff format --check .
uv run pyright src tests scripts
uv run pytest
```

For changes that touch the STAC catalog:

```bash
uv run stac-validator validate stac/catalog.json --recursive
```

For changes that touch the asset GeoParquet schema:

```bash
uv run python -m wildfire_exposure_eo.cli validate-schema outputs/parquet/<latest>.parquet
```

If any of these fail, the task is not complete. Do not push, do not declare done.

## Verify-then-act protocol

Before running compute on the full AOI, prove the path works on a 1 km × 1 km tile (`data/aoi/smoke.geojson`). The CLI exposes `--smoke` for this. The session log must show the smoke run before the full run.

Before loading STAC items:

1. List candidates with `pystac-client.search(...).items()` and log every returned `id`.
2. Apply deterministic ordering (`sort by datetime, tie-break by id`).
3. Then load.

Skipping step 1 is a bug.

## Schema-validate every artifact

Every artifact this repo produces has a schema. Schemas live in `src/wildfire_exposure_eo/schemas/` as Pydantic v2 models or JSON Schema files. The expected pattern:

```python
from wildfire_exposure_eo.schemas import ScoredAsset
ScoredAsset.model_validate(row)  # raises on bad row
```

CI runs schema validation on a sample of every output committed. Adding an output without a schema is a bug.

## Fact-checking checklist for any narrative claim

When the README, `validation_report.md`, or commit message contains a numeric or factual claim, that claim must be reproducible from a script in this repo. Each session must:

1. Identify every numeric / factual claim added in the change.
2. Locate the script that produces it.
3. If the script does not exist, write it under `scripts/` and reference it from the doc.
4. Cite the script in the doc as `<!-- generated by: scripts/04_validate.py at <commit> -->`.

If a claim cannot be reproduced from this repo (e.g., an external statistic), it must be cited inline with a URL and a date-of-access.

## Test gates

| Gate | Location | What it checks |
|---|---|---|
| `tests/unit/` | per-module | pure-function correctness |
| `tests/integration/` | per-pipeline-stage | with small fixtures, end-to-end of a stage |
| `tests/golden/` | frozen outputs | regression against stored expected outputs |
| `tests/schemas/` | Pydantic models | every schema parses representative examples |
| Property tests | `hypothesis` | geometry transforms, CRS round-trips |
| Smoke run | `--smoke` flag | full pipeline on 1 km × 1 km tile |

A new feature ships with at least: one unit test, one schema test (if a schema changed), and a smoke-run check.

## Working with prompts/

The `prompts/` directory contains canonical Claude Code prompts for major work-units. When starting a session for one of those work-units:

1. Open the prompt file, read it end-to-end.
2. Confirm the prerequisites listed at the top are met.
3. Execute. Do not deviate from the listed deliverables without flagging it.
4. At the end, leave a session note in `prompts/_session_log.md` with the date, prompt name, commits produced, gates passed, gates skipped (if any).

## TodoWrite

Use TodoWrite for any task with three or more steps. The user has it pinned to a widget; it is not optional cosmetic. Mark items in_progress and completed in real time.

## Don't surprise the human

Anything that touches:

- the repo's public surface (README headlines, repository description, license),
- the model interface (CLI arguments, public function signatures),
- the data contract (schemas, file formats, output paths),
- the pinned dependency list,

requires explicit human approval in-session before it ships. Surface it as a question, not a fait accompli.

## Common failure modes (anti-patterns to refuse)

- *"Let me just hardcode this AOI for now."* — No. Edit `data/aoi/pilot.geojson`.
- *"I'll use Shapefile for this output, it's simpler."* — No. GeoParquet.
- *"This score correlates with risk so let's call it a probability."* — No. Read non-negotiable #6.
- *"Prithvi burn-scar inference is essentially ignition prediction."* — No. We detect burn *scars* — visible post-event spectral signatures of fires that already happened. Ignition prediction is a forecasting task this project does not perform. Phrase carefully in README, docs, and any external write-up.
- *"I'll skip pyright on this file, it has too many errors."* — No. Fix or `# pyright: ignore[<reason>]` per line with a justification comment.
- *"Let me add `<heavy-dep>` to make this faster."* — Pause. Justify in writing first.
- *"I'll commit a generated artifact under `data/`."* — No. `data/` is inputs only. Generated files go to `outputs/` (gitignored except for STAC catalog under `stac/`).

## Session boot checklist (paste into the first message of any new CC session)

```
- [ ] Read README.md, CLAUDE.md, PRE_DEV_CHECKLIST.md.
- [ ] Confirm `uv sync --locked --extra dev` succeeds (bare `--locked` removes the dev extras — ruff/pyright/pytest live in `[project.optional-dependencies].dev`).
- [ ] Confirm `uv run pytest -q` is green on a clean checkout.
- [ ] Confirm `data/aoi/pilot.geojson` exists and loads.
- [ ] Run `uv run wildfire-exposure-eo audit` — fix any RED rows before proceeding.
- [ ] Open the relevant `prompts/*.md` for this work-unit.
- [ ] Use TodoWrite to plan the session.
```

## Escalation

If a non-negotiable seems to be blocking real progress, the right move is to surface it explicitly to the human, not to work around it. Frame it as: "Constraint X is blocking Y because Z; the alternatives I see are A, B, C; my recommendation is B because…" Then wait.
