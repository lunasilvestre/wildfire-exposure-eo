## Purpose

Produce `docs/validation_report.md` — lift and Spearman of the exposure rank
against subsequent ICNF burns under the §12 temporal-leakage rule, with the
mandatory burn-history ablation — plus the demo-path documentation, and a
**proposal** (not a commit) for the README close-out. The defensible claim,
verbatim from the plan: *"a transparent, reproducible prioritization screen
validated against subsequent burns"* — nothing stronger.

## Prerequisites

- [ ] WU-6 shipped; a **backdated** pilot scoring run works
      (`score --window-end 2024-12-31`).
- [ ] WU-4 burns parquet spans vintages strictly after 2024 (the 2025 fires
      are present — WU-1's sanity check used three of them). Confirm by
      query before anything else; if no post-window vintages exist, HIL.
- [ ] Read `CLAUDE.md` (#6, #9: no production claims) and
      `docs/methodology.md` §12 + the close-out plan's four methodological
      caveats — they are requirements here, not background.

## Deliverables

1. **`src/wildfire_exposure_eo/validation.py`**:
   - `assert_no_temporal_leakage(score_window: DateRange, burns: gpd.GeoDataFrame)`
     — exactly the §12 hard rule; raises unless every validation burn year >
     `score_window.end.year`. Unit test + `hypothesis` property test
     (random window/year combos; assertion fires iff leakage).
   - `asset_burn_labels(assets, burns, *, years: list[int]) -> pd.Series` —
     boolean: asset buffer intersects any validation-year burn polygon
     (EPSG:32629 overlay).
   - `lift_table(scores, labels, *, deciles=10)` and
     `spearman_rank(scores, labels)` — numpy/scipy-only, no new deps.

2. **`scripts/11_validate.py`** — runs the full evaluation and **emits the
   markdown** of the results section verbatim (the WU-1 audit-script pattern:
   every reported number is script output). Inputs: the backdated exposure
   parquet (window-end 2024-12-31), burns parquet, validation years = all
   vintages > window-end. Outputs: lift table + curve data, Spearman with
   p-value, n-assets, n-burned, base rate, and the **ablation pair**: the
   identical evaluation for a score variant recomputed without
   `historical_burn_share` and `recent_burn_share_12mo` (weights
   renormalised for the variant only — it is an analysis artefact, never
   shipped as product output).

3. **`docs/validation_report.md`** with, at minimum:
   - The leakage rule, the window (inputs ≤ 2024-12-31), validation years,
     and which features were active in the validated configuration —
     state explicitly that `recent_burn_share_12mo` (and FWI, if dropped in
     WU-6) was **not** active in the backdated run and why.
   - Lift + Spearman, with and without burn-history features, plainly
     comparing: *which features carry the signal* (plan caveat #1). State
     that spatial autocorrelation of fire means lift flatters the score and
     the leakage split is necessary but not sufficient.
   - Fuel scale-mismatch paragraph quoting the WU-5 sidecar's effective
     EFFIS resolution (plan caveat #3); never imply parcel-level precision.
   - The §11 scope boundary: AOI-relative ranks, not cross-AOI comparable.
   - Rank-not-probability language throughout. Brier score: **omit** — it
     presumes a probability forecast; cite `exposure_score.yaml`'s
     `calibration.method: report_only` and note the omission explicitly so
     the YAML's metric list and the report don't silently disagree.
4. **Demo path** — `docs/demo.md` (or README section in the proposal):
   the smoke-AOI sequence `audit → fetch-osm → fetch-rasters → fetch-burns →
   fuel-layer → score → validate`, CPU-only, pre-baked WU-1 burn-scar COG,
   with **measured** wall-clock per step (run it; paste real numbers; §14
   budget is ≤ 30 min total).

5. **README close-out proposal — do not edit README.md.** Write
   `docs/_readme_closeout_proposal.md` containing the exact diff-ready text:
   status section "Pre-development" → dated "Demonstrator complete — scope"
   with the civic-tech framing (municípios / civil-protection / open-data
   audience first; no vendor adjacency beyond one related-work line), the
   validation headline numbers (script-generated), and a placeholder section
   for the WU-8 figures embed. Then write `prompts/_HIL.md` asking Nelson to
   approve (a) the README status text and (b) the upcoming WU-8 figure-embed
   section, commit both, and **end the session** — the driver halts there by
   design (public surface, CLAUDE.md "Don't surprise the human").

6. **`prompts/_session_log.md`** entry.

## Constraints

- Every number in the report reproducible by `scripts/11_validate.py`;
  the generated-by comment cites the emitting commit.
- No probability, risk-probability, chance-of-fire, or production-readiness
  language anywhere (#6, #9). The words *screening rank* appear in the
  report's first paragraph.
- The ablation variant's parquet goes under `outputs/` (gitignored), never
  `stac/` or committed outputs — it is not a product artefact.
- If lift is ≤ 1.0 or Spearman ≈ 0 for the no-burn-history variant, report
  it plainly — a validation section that hides this is marketing (plan
  caveat #1). Do not tune weights to improve the numbers in this WU.

   burned-buffer assets.
