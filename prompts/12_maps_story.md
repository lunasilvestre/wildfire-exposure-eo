## Purpose

Produce the visual deliverables: five static figures under `docs/figures/`
plus **one** self-contained interactive HTML map, all generated by a script
under `scripts/` (never hand-made), embedded in the README per the approved
proposal.

## Pre-approved decisions (Nelson, 2026-06-11 checkpoint)

- **New dependencies**: `folium` (pinned) for the HTML map and
  `matplotlib` (pinned, if not already present) for static figures —
  justification: lightest mature options; lonboard rejected (heavier GPU/
  jupyter stack for no benefit at 3 k assets). Both land in
  `[project.optional-dependencies].viz` with one-line justifications in the
  PR description.
- **Basemap/imagery**: the S2 true-colour composite is built from the same
  deterministic STAC machinery (median of the most recent low-cloud spring
  window items; IDs logged). For the HTML map, OSM raster tiles are an
  acceptable basemap (attribution required); do not bundle proprietary
  basemaps.

## Deliverables

1. **`scripts/12_make_figures.py`** — single entry point, `--smoke` flag,
   deterministic (seed 42, fixed figure size/dpi, sorted draw order).
   Generates:
   1. `fig1_exposure_map.png` — assets coloured by `exposure_rank`
      (perceptually uniform colormap, e.g. viridis) over the S2 true-colour
      composite; AOI outline; scale bar; legend labelled "exposure rank
      (relative, AOI-normalised)".
   2. `fig2_fuel_layer.png` — WU-5 fuel classes with the crosswalk class
      names in the legend; effective-resolution note in the caption.
   3. `fig3_burn_scar.png` — WU-1 burn-scar composite over the AOI with the
      0.5 threshold contour; caption carries the max-composite caveat.
   4. `fig4_icnf_overlay.png` — exposure map dimmed, ICNF validation-year
      perimeters outlined on top (the "did the rank point at the right
      places" picture).
   5. `fig5_lift_curve.png` — from WU-7's script outputs (re-emit via
      `scripts/11_validate.py` data, never re-derived numbers).
2. **`docs/figures/exposure_map.html`** — folium, self-contained
   (`--no-cdn` inline assets if available; otherwise document the CDN
   dependency in a comment), layers: ranked assets (colour + popup with
   asset_id, class, rank, top-3 contributing features), fuel layer toggle
   (image overlay), ICNF perimeters toggle. File size sanity: < 25 MB.
3. **README embed** — apply the approved figure section from the WU-7
   proposal: `fig1` near the top, the HTML map linked (GitHub won't render
   it inline; link via raw/HTML-preview note), `fig5` in the validation
   section. No other README edits.
4. **Tests** — unit: figure script's data-loading helpers; smoke: `--smoke`
   produces all six artefacts non-empty; CI does **not** regenerate pilot
   figures (committed artefacts; regeneration documented in the script
   header).
5. **`prompts/_session_log.md`** entry.

## Constraints

- Figures are *generated artefacts that we commit deliberately* (the one
  exception to "no generated files in git" — they are documentation).
  `docs/figures/` only; nothing under `data/` or `outputs/`.
- Every caption that mentions the score says **rank**; the words
  "probability" and "risk" do not appear in any figure, legend, caption,
  alt-text, or popup.
- Attribution: Sentinel-2 (ESA Copernicus), OSM contributors, ICNF, EFFIS,
  DGT, ETH GCH — one attribution block in the README figure section and in
  the HTML map footer.
- Colour accessibility: one colormap, perceptually uniform, no red/green
  encoding of rank.

## Test gates

The four standard gates plus `uv run python scripts/12_make_figures.py
--smoke`. (`stac/` untouched → no stac-validator needed; run it anyway if
anything under `stac/` changes.)

## Verification

1. Open every PNG; confirm legends, captions, attribution, no placeholder
   text. Open the HTML map locally; toggle all layers; click 3 popups.
2. README renders correctly (GitHub markdown preview via `gh` or pushed
   branch view) — fig1 visible above the fold.
3. Grep the artefacts' captions/popups for "probab"/"risk" — zero hits.

## Out of scope

- Dashboards, servers, tiling services; the HTML file is static.
- Any re-scoring or validation changes; figures consume committed outputs.
- Social-media/marketing imagery.

## Done when

applied per the approved proposal; session log updated; pushed with CI
green — and with it, the project's Definition of DONE visual clause is met.
