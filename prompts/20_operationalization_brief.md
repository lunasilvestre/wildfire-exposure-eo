## Mission

Turn the ranked GeoParquet into a **decision product**: a "top-N critical assets
to prioritise this season" brief with **absolute, actionable thresholds** — not
just an AOI-relative rank and a dev-oriented map. This is what makes the project
an *operational-planning* pilot rather than a methodology demo. Its credibility
rests on robustness (pillar 2) + freshness (pillar 0), so this runs after both.

**Scope guard (#6, #9).** A *prioritization screen*, not a forecast and not a
production service. The brief recommends an **inspection/clearing priority order**,
explicitly framed as a screen "validated against subsequent burns" — never a
probability of fire, never "operationally validated" or "production-ready".

## Phase 0 — The absolute-threshold problem (design decision, resolve first)

The score is **AOI-relative** by design (percentile-rank within AOI), but the
brief promises **absolute** thresholds. These conflict. Resolve and document one
of:

1. **Raw-feature thresholds** — define actionable bands on the pre-percentile
   feature space (e.g. "canopy p90 ≥ X m AND fuel-severity ≥ Y AND fire-danger ≥
   Z"), so the threshold means the same thing in any AOI.
2. **National reference distribution** — build the cross-AOI reference
   strategy.md §7(6) flags, and define absolute bands against it.
3. **Hybrid** — top-N per AOI by rank (the relative part) + an absolute "minimum
   actionability floor" on raw features (so a top-N asset in a low-exposure AOI
   isn't surfaced if it's below the floor).

Pick one, justify it, and record the decision in the session log + the brief's
methods note. **Do not silently relabel AOI-relative ranks as absolute** — that
would misrepresent the score (#6).

## Phase 1 — Decision-brief generator

- New `scripts/20_make_decision_brief.py` (seed 42, #4) that reads the scored
  GeoParquet and emits, per AOI / per município:
  - a **top-N critical-assets table** (default N configurable) with: asset id +
    class, exposure rank, the absolute band/floor result from Phase 0, the
    criticality weight, the seasonal fire-danger value (pillar 0), and the
    topology context (pillar 1, if present: "feeds N downstream assets").
  - a one-line **action** per asset in screening language ("prioritise fuel
    clearing in buffer", "schedule line inspection") — never a guarantee.
  - per-row provenance retained (it derives from the parquet, which carries it).
- Output formats: a committed **Markdown** brief under `docs/` (or
  `outputs/briefs/`) **and** the underlying GeoParquet slice (STAC-native /
  GeoParquet only, #5 — no PDF as the *primary* artefact, though a rendered PDF
  may be a convenience copy). Every number script-generated with the
  `<!-- generated by: scripts/20_make_decision_brief.py at <commit> -->` citation.

## Phase 2 — Honest framing + a planner-readable figure

- A figure (via `scripts/`, not hand-made): top-N assets on satellite imagery
  with the absolute-band colouring, so a planner sees the prioritised assets at a
  glance.
- A methods box in the brief restating the scope boundaries verbatim from
  `docs/operationalization.md` §2 (rank not probability, AOI-relative caveat and
  how Phase 0 handled it, demonstrator not service).
