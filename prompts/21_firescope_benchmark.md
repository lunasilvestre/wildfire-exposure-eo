## Mission

Benchmark the repo's transparent per-asset exposure rank against **FireScope**
(INSAIT, CVPR 2026, arXiv:2511.17171 — public Europe-wide 30 m wildfire **risk**
raster, CC-BY-4.0): consume their open raster through the **identical**
leakage-clean WU-7 harness, report agreement vs the transparent rank **in both
directions**, and **optionally** add FireScope-zonal as a candidate score feature
with an ablation row. Strong portfolio signal: "I can benchmark my transparent
screen against a CVPR model and say honestly where each wins."

**Hard exclusion:** NO MEJOR / LUCI (proprietary — breaks open-reproducible). NO
training, NO fine-tuning, NO model execution — consumption of their published
raster only.

## Phase 0 — Downloadability + license gate (run in Wave 1; this WU is GATED on it)

The HF dataset `INSAIT-Institute/firescope-risk-2026` had an **empty card**,
`imagefolder` packaging, ~12.3 GB, broken viewer; `firescope.ai` 403'd a bot.
Before any benchmark design:

- List the HF repo file tree (`huggingface_hub` API). Determine: actual contents
  (georeferenced rasters? tiles? CRS? resolution? vintage), whether parquet
  sidecars exist, and the **license** as published.
- **Decision rule (from `prompts/13`):** if parquet exists → consume remotely via
  DuckDB (`httpfs`/`hf://`) without a full download; else → plan an atlas download
  (~12.3 GB, sha256 manifest) + an explicit-CRS reader. If the dataset does **not**
  contain a usable Europe risk raster, **STOP** and report — check their GitHub
  release / paper artefacts as the alternative; do not improvise.
- License must be confirmed CC-BY-4.0 (or otherwise redistribution-compatible for
  *derived comparison artefacts only* — no raw raster committed to this repo).

Deliverable: `scripts/21_firescope_feasibility.py` →
`outputs/diagnostics/21_firescope_feasibility.json` (file tree summary, CRS/res/
vintage if determinable, license, parquet-present flag, GO/NO-GO verdict).

**If NO-GO: park pillar 4, log the reason, do not force it.** The rest of the
program is unaffected (pillar 4 is a credibility add-on, not a dependency).

## Phase 1 — Reader + alignment (only if Phase 0 = GO)

- New `src/wildfire_exposure_eo/firescope.py` — reader + **explicit-CRS**
  alignment of the FireScope raster to our grids (#2); provenance (dataset
  revision sha, file shas, license note, #1).

## Phase 2 — Asset-level comparison (pilot AOI)

- Zonal FireScope risk per asset buffer → percentile rank → Spearman +
  rank-agreement vs our exposure rank; top-decile overlap table; per-class
  breakdown. **Vocabulary firewall (#6):** FireScope outputs are "risk" *in their
  terminology* (quoted, attributed); ours stay *exposure rank*; the comparison
  never converts our rank into a probability.

## Phase 3 — Head-to-head historical validation (the killer comparison)

- Both rankers through the **identical** WU-7 leakage-clean harness (backdated
  window, post-window ICNF vintages): lift + Spearman **side by side**, on the
  pilot AOI and — after pillar 2 lands — the **widest universe** the data supports
  (the new AOIs make this far more legible than at sketch time). Where the wide
  run is raster-vs-burns (assets only exist for the modelled AOIs), say so plainly.

## Phase 4 — Optional: FireScope-zonal as a candidate score feature

- Optionally add the per-asset FireScope-zonal value as a **candidate** feature
  and report an **ablation row** (score with vs without it) in the pillar-2
  combined validation. If integrated into the weighted score:
  `config/exposure_score.yaml` weight edit is **SERIALIZED** (coordination point
  with pillars 0/1 — land it alone, re-normalise to 1.0). Default: report as a
  candidate + ablation first; integrate only if it carries signal.

## Phase 5 — Report

- `docs/firescope_comparison.md` + figures via `scripts/21_*.py` (every number
  script-generated, `<!-- generated by -->` citation): delta maps (our rank −

  written **verdict** honest in both directions. CC-BY-4.0 attribution wherever
  their data appears. **No redistribution of their raster** — derived comparison
  artefacts only.
